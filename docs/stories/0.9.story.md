## Story

**Como** owner do projeto,
**eu quero** que a página `/stadiums` reflita os **16 estádios oficiais** da Copa do Mundo FIFA 2026 com nomes corretos, **capacidades reais**, **ano de inauguração** e **imagens reais** (não placeholder/aleatórias),

Frontend não precisa ser tocado — `/api/stadiums` já é consumido por `Stadiums.tsx`. Tudo é refletido via dados.

## Dev Notes

**Por que não criar nova coluna `inauguration_year`?**
- Schema `stadiums` já tem `description` (NVARCHAR(MAX)). Adicionar coluna exige migration de schema + atualizar API + atualizar frontend para mostrar campo.
- Para evento educacional, ano + fato em description é suficiente e didático ("Inaugurado em 2017. Casa do Atlanta Falcons.").
- Tech debt: TD-14 (criar coluna estruturada se houver demanda).

**Por que MERGE em vez de DELETE+INSERT?**
- Tabela `stadiums` é referenciada por `matches.stadium_id` (FK). DELETE quebraria matches existentes.
- MERGE preserva IDs INT IDENTITY dos 9 existentes (matches do bacpac continuam ligados).

**Wikipedia Commons URLs:**
- Best-guess. Se algumas falharem, @dev valida e substitui.
- Padrão de URL do Wikipedia: `upload.wikimedia.org/wikipedia/commons/thumb/{hash1}/{hash2}/{filename}/800px-{filename}` — os hashes são determinísticos (MD5 do filename) mas vêm da pesquisa.
